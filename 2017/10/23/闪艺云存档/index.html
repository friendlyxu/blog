App云存档

  1. 数据库分表根据uid哈希分1024张表，再哈希分32个库
  2. 根据产品提供一年半后的数据量，每天活跃用户值，存储预计单表存储百万数据量
  3. 缓存根据哈希一致性算法引入第三方库，通过uid分布到不同的redis实例，存储一个月的热数据处理
  4. 存档统计模块，通过redis缓存队列，通过cron脚本同步到数据库永久保存

    流程图

    st=>start: 引入云存档主要类库，配置
    op_init_obj=>operation: 初始化类对象，接收用户登录信息
    cond_user_info=>condition: 是否存在用户uid，判断登录
    op_get_param=>operation: 接收client_type客户端类型参数，gid游戏ID
    cond_gid=>condition: 游戏ID是否为空
    op_end_err=>end: 输出错误信息
    op_end=>end: 请求结果
    op_switch_start=>operation: ac判断switch_start
    cond_ac_save=>condition: ac=save?
    cond_ac_info=>condition: ac=info?
    op_switch_end=>operation: ac判断switch_end
    op_content=>operation: 接收存档内容
    cond_content_empty=>condition: 存档内容是否为空
    cond_content_check=>condition: 检查存档内容是否符合格式
    op_progress=>operation: 接收进度值参数
    op_init_db_arr=>operation: 初始化导入表的数组字段
    op_save=>operation: 保存存档
    cond_save_result_1=>condition: 保存返回1？
    cond_save_result_2=>condition: 保存返回2？
    op_statistic_1=>operation: 保存操作相关的存档统计
    op_statistic_2=>operation: 更新操作相关的存档统计
    op_fetch=>operation: 获取存档
    cond_fetch_content=>condition: 获取内容是否为空
    op_decrypt=>operation: 解密存档内容
    op_statistic_3=>operation: 获取操作相关的存档统计

st->op_init_obj->cond_user_info
cond_user_info(yes)->op_get_param->cond_gid
cond_gid(yes,right)->op_end_err
cond_gid(no)->op_switch_start->cond_ac_save
cond_ac_save(yes)->op_content->cond_content_empty
cond_content_empty(yes)->op_end_err
cond_content_empty(no)->cond_content_check
cond_content_check(yes)->op_progress->op_init_db_arr->op_save->cond_save_result_1
cond_save_result_1(yes)->op_statistic_1->op_end
cond_save_result_1(no)->cond_save_result_2
cond_save_result_2(yes)->op_statistic_2->op_end
cond_save_result_2(no)->op_end
cond_content_check(no)->op_end_err
cond_ac_save(no)->cond_ac_info
cond_ac_info(yes)->op_fetch->cond_fetch_content
cond_fetch_content(yes)->op_end_err
cond_fetch_content(no)->op_decrypt->op_statistic_3->op_end
cond_ac_info(no)->op_switch_end
cond_user_info(no)->op_end_err

  1. 保存更新接口

    /**

    • 存档存储,带有更新
      *
    • @param int $uid 用户ID
    • @param int $gid 游戏ID
    • @param array $data 存储字段数组
    • @return bool|int 1插入成功 2更新成功
      */
      public function save($uid, $gid, $data)
      接口名：云存档保存更新接口
      App请求地址：http://app.3000.com/?m=cloud_save&op=index&ac=save&client_type=1&v=1.0.0
      Pc请求地址：http://my.3000.com/?m=cloud_save&op=index&ac=save&client_type=1&v=1.0.0
      请求方式：post

      请求参数

      | 参数名 | 类型 | 必填 | 描述 | 默认值 |
      | ——– | —— | —- | ——————————— | ———— |
      | u | string | 是 | 用户登录参数 | null |
      | token | string | 是 | 用户登录参数 | null |
      | content | string | 是 | 存档内容JSON字符串，base64编码 | eyJzcyI6MX0= |
      | gid | int | 是 | 游戏ID | 1 |
      | progress | int | 否 | 存档进度数量, 新增数量, 第一次为存档总进度，之后请求都是增量数 | 1 |
      | sign | string | 是 | 签名，content，base64编码 | xx |
      | time | int | 是 | 时间戳, 300秒有效 | 1487670893 |
  2. sign: eg.md5(md5(‘content=eyJzcyI6MX0=&gid=3&progress=1&time=1487664238&v_key=shanyi_1.0.0**’)), app版本区分于pc版本，api_v改成aes_key

    返回示例

返回参数

  1. msg:保存成功/更新成功/操作失败

 */
public function fetch_archive($uid, $gid, $field = 'content')
接口名：云存档信息接口
App请求地址：http://app.3000.com/?m=cloud_save&op=index&ac=info&client_type=1&v=1.0.0
Pc请求地址：http://my.3000.com/?m=cloud_save&op=index&ac=info&client_type=1&v=1.0.0
请求方式：post

请求参数

参数名 类型 必填 描述 默认值
u string 是 用户登录参数 null
token string 是 用户登录参数 null
gid int 是 游戏ID 1
sign string 是 签名 xx
time int 是 时间戳, 300秒有效 1487670893
  1. sign: eg.md5(md5(‘content=eyJzcyI6MX0=&gid=3&progress=1&time=1487664238&v_key=shanyi_1.0.0**’)), app版本区分于pc版本，api_v改成aes_key

    返回示例

返回参数

  1. content 存档内容，base64编码格式, 无效游戏ID或者存档为空，返回空字符串
